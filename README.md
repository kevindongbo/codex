# 东铂跨境 · 轻量跨境 ERP

面向跨境电商团队的浏览器端运营工作台，整合商品主档、采购在途、仓库库存、订单出库和竞品监控。项目为纯静态网页，可部署到 GitHub Pages，也可以直接打开 `index.html` 使用。

## 三个独立模块

- **商品中心**：统一维护本店商品、直接竞品和间接竞品；每条商品链接必须配置图片。本店商品额外维护唯一 SKU、标准采购成本、安全库存、供应商和采购链接。
- **仓配中心**：包含“采购与在途”“库存管理”“订单与出库”三个二级页面。采购单产生在途，确认收货才进入已在库；订单先整单锁定库存，完成拣货、复核和出库后才扣减已在库。
- **竞品监控**：保存买家前台公开价格、累计销量、评分与评价快照，计算相邻快照变化，并提示降价、销量回退、低星评价和长时间未更新。

## 库存计算口径

```text
在途 = 有效采购数量 - 已收货数量 - 已取消数量
已在库 = 期初 + 收货 + 盘盈 - 出库 - 盘亏 - 报损
锁定 = 尚未出库或取消的订单预留数量
可用 = 已在库 - 锁定
```

草稿采购单不计入在途。订单库存不足时不会部分锁定；取消订单会释放锁定，确认出库会同时减少已在库和锁定。所有库存变化都会生成不可直接编辑的库存流水。

## 数据保存与迁移

业务数据保存在当前浏览器的 `localStorage` 中，不会上传到服务器。新版会自动识别旧版 `dongbo-crossborder.v1` 和 `pulsetrack.manual.v2` 数据，将旧版已在库转换为期初库存，将旧版在途转换为待核对的迁移采购单。

“导出数据”当前导出全部商品主档和每个商品的最新竞品快照；采购单、订单和完整库存流水仍保存在当前浏览器中。正式多人使用前，建议接入后端数据库、登录权限、操作审计和完整 JSON 备份。

## 本地开发

```bash
npm run build
npm test
```

构建产物位于 `dist/server/index.js`。网页本身不依赖框架或第三方前端库。

## 当前 MVP 边界

- 单浏览器、单用户、单默认仓。
- 采购、订单与快照以手工录入为主，尚未连接 TikTok Shop、Shopee、Temu 或 Ozon API。
- 成本金额按币种分别展示，不会在没有汇率的情况下直接混加。
- 已出库订单不能直接取消，可按原订单明细办理退货入库；换货、退款和质检状态留待后续版本。
